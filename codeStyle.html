0. Directory Structure

All folder names should be in simple-lowercase alphabets without camelcase or hyphenation words. 


                        root
                          |-styles
                          |   |-lib
                          |   |  |-css
                          |   |  |  |-CSS FRAMEWORK CSS FILES
                          |   |  |-fonts
                          |   |     |-CSS FRAMEWORK FONT FILES
                          |   |-dev
                          |      |-css
                          |         |-DEVELOPER DEFINED CSS FILES
                          |-scripts
                          |   |-lib
                          |   |  |-js
                          |   |     |-JS FRAMEWORK FILES
                          |   |-dev
                          |      |-controllers
                          |      |     |-DEVELOPER DEFINED JS FILES
                          |      |-services
                          |      |     |-DEVELOPER DEFINED JS FILES
                          |      |-directives
                          |            |-DEVELOPER DEFINED JS FILES
                          |-pages
                              |-USER DEFINED HTML PAGES

0.1. Files

  • Naming convention for HTML, CSS and JS files should be in CamelCase, for example, FileOne.html and not fileOne.html
  • Every HTML page should have its own CSS file, for example, FileOne.html should have FileOne.css and similarly, for other pages. Although, when styles start to overlap among different pages then, one reusableStyles.css file should be created and those styles should be added to it.

2. Editor Configuration

Best way to ensure that every team member's code looks alike, in terms of indentation, code-markup, styles, etc; is to use a editor configuration tool like EditorConfig. Following is a sample .editorconfig file that can be used for most of the HTML, CSS, JS projects. 


                            root = true
                            [*]
                            charset = utf-8
                            indent_style = space
                            indent_size = 4
                            trim_trailing_whitespace = true
                            end_of_line = lf
                            insert_final_newline = true
